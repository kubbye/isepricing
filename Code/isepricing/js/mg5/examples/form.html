Menu G5 Examples: The revenge of menus

yxScripts.com

 

The form select list has been a headache to dhtml menu users for a long time. The menu users have to either put the select list and the menu away from each other, or put the form in a layer and use some show/hide layer tricks.

Started with the Gecko browsers then followed by Opera7, Safari and IE5/Mac, the form select list would no longer stay on top of dhtml layers. However, the mainstream IE browsers don't seem to be taking any move on this issue, which might give you one more reason to hate M$.

Well actually, for IE5.5 and IE6, the IFRAME element will now step over the form select list and can be z-indexed below a dhtml layer, thus it can be used to make a workaround.

Menu G5 has already implemented this workaround for you and you can be happy again with the form select list.
